3.7

位运算

>> ：右移运算符，num>>1 相当于 num 除以 2

<< ：左移运算符，num<<1 相当于 num 乘以2

>>> ：无符号右移

无符号右移的规则只记住一点：忽略了符号位扩展，0补最高位 无符号右移运算符 >>> 只是对32位和64位的值有意义。

这一题用动态规划进行求解是真的巧妙

首先我们设数组dp[left][right] 是从left到 right 的一段回文字符串，

那么若 dp[left+1][right-1] 是回文字符串，那么dp[left][right] 是回文字符串的条件就是

dp[left][right]=dp[left+1][right-1]&& s.chatAt(left)==s.charAt(right)

3.11

今天碰到了一个链表的小坑，如果链表的 node.next == null ，这里指向了就等于封闭了，没有下一个元素的地址了，如果这个时候 node = node.next 的话，那么就不能将其他链表挂到 node 上了，因为node.pre无法 找到它的地址，毕竟node.pre.next已经封闭了。

正确的做法应该是在 node.next ==null 时，如果想挂其他链表就马上把它的 next 挂到 node.next 上，这样就不会出现找不到地址的情况。
