4.22

寻找旋转排序数组中的最小值

前缀和

前缀和：nums 数组的第 0 项到 当前项 的和。
首先定义 prefixSum 数组，prefixSum[x]：第 0 项到 第 x 项 的和。
prefixSum[x] = nums[0] + nums[1] + ··· +nums[x]

那么就可以得到如下的一些推论：

nums 的某项 = 两个相邻前缀和的差：
nums[x] = prefixSum[x] - prefixSum[x - 1]

nums 数组的第 i 到 j 项的和为：
nums[i] + ··· + nums[j] = prefixSum[j] - prefixSum[i - 1]

当 i 为 0，此时 i - 1 == -1，我们故意让 prefixSum[-1] = 0，使得通式在 i = 0 时也成立：
nums[0] + ··· + nums[j] = prefixSum[j]，而由于数组下标从0开始，所以我们令prefixSum[0] = 0，

然后 prefixSum[x + 1] = nums[0] + nums[1] + ··· +nums[x]

和为 K 的子数组

这道题就可以利用前缀树组进行求解

1、直接将问题转化成，使prefixSum[j] - prefixSum[i - 1] == k

2、通过哈希表进行优化

因为我们不需要 prefixSum 数组的具体实现，只需要统计从0到当前位置前缀和的值出现的次数就行.

凸包问题

andrew算法

1、对所有点进行双关键字排序，先根据 x 坐标排升序，再根据 y 坐标排升序

根据 x 坐标排升序的目的：能够往一个方向画出凸包边缘（从左往右画出一半，从右往左画出另一半）

根据 y 坐标排升序的目的：确保一旦我们现在从 a 到 b 进行连线，那么 a 到 b 之间的所有点都能够确保被围住

2、使用栈来维护所有凸包上的点，或者说凸包上的边更为准确（因为凸包起点元素会在栈中首尾出现两次），栈中相邻元素代表凸包上的一条边；

3、分别从前往后和从后往前处理排序好的所有点，来分别画出凸包的上下两部分，根据画的是第一部分还是第二部分，维护栈内元素的处理逻辑稍有不同：

  • 第一部分：
    • 若栈内元素少于 2 个，组成一条线至少需要两个点，说明此时第一条边都还没画出，直接将元素添加到栈中；
    • 若栈内元素不少于 2 个，考虑是否要将栈顶的边删掉（由栈顶前两个点组成的边），假设栈顶元素为b，栈顶元素的下一位是 a，即栈顶存在一条 a 到 b 的边，当前处理到的点为 c，此时根据 ac 边是否在 ab边的逆时针方向，来决定是否将 ab 边去掉（一二部分判断顺序统一即可），这就可以通过叉乘来进行判断，正逆负顺
  • 第二部分：唯一要注意的是，第一部分的凸包边我们不能删去，假设处理完第一部分凸包，我们栈内有 m 个元素，我们需要将上述栈顶顶元素不少于2个的逻辑替换为栈顶元素大于 m 个，同时已参与到凸包第一部分的点，不能再考虑，因此需要额外使用一个 visited 数组来记录使用过的点

5.6

找到 K 个最接近的元素

普通双指针方法就不写了，很简单

主要介绍双指针+二分查找

基本思路：通过二分来确定这 K 个元素左边界，然后就能通过 left + k 找到这 K 个元素

首先找到二分查找的左右边界值 left = 0，right = arr.length - k

我们所要找的目标值就是K个元素左边界值就在这里面

找临界情况，也就是左边界 L 和 x 应该满足的关系

|x - arr[L]| >= |arr[L+k] - x|

由于是数组是递增的，所以去掉绝对值后就是x - arr[L] >= arr[L+k] - x

通过mid是否满足临界值来决定左右指针的移动，

临界情况的左半部分一定不满足条件，右半部分一定满足条件

那么如果mid值在左半部分，也就是 x - arr[mid] < arr[mid + k] - x 时，左边的值一定不存在边界条件

所以左指针向右移动到 mid + 1 位置，而同理得到 右指针 向左移动到 mid 位置

ok，左右指针移动问题解决，思路基本梳理完成~

袋子里最少数目的球

这一题的转化非常巧妙

将原问题转化成给定 maxOperations 次操作次数，能否可以使得单个袋子里球数目的最大值不超过 y

如果单个袋子里有 x 个球，那么操作次数就是 $\lfloor\dfrac{x-1}{y}\rfloor$

那么只要使的 y 满足 $\sum\lfloor\dfrac{x-1}{y}\rfloor\leqslant$ maxOperations 就行

这里的最大数目就从 1 — max(nums) 中进行二分查找，

临界情况就是 $\sum\lfloor\dfrac{x-1}{y}\rfloor=$ maxOperations

5.10

猫和老鼠

暴力搜索超时

所以使用记忆化搜索，定义一个状态数组 f[k][i][j]

f[k][i][j] 为当前进行了 k 步，老鼠所在位置为 i，猫所在位置为 j 时，最终获胜的情况

如果 i = 0，老鼠进洞获胜，f[k][i][j] = 1

如果 i = j，猫和老鼠在同一位置，猫获胜，f[k][i][j] = 2

下面就是平局情况，需要一些讨论

如果当前 k 为偶数，那么就是老鼠移动，此时所能转移所有点为 f[k+1][next][j] ，采取最优移动策略，那么 f[k][i][j] 会优先往 f[k+1][next][j] = 1 移动，其次往 f[k+1][next][j] = 0 移动，最后才往 f[k+1][next][j] = 2移动

如果当前 k 为奇数，那么就是猫移动，此时此时所能转移所有点为 f[k+1][i][next] ，采取最优移动策略，那么 f[k][i][j] 会优先往 f[k+1][i][next] = 2 移动，其次往 f[k+1][i][nex]t = 0 移动，最后才往 f[k+1][i][next] = 1移动

因为可以发现使用状态数组，因为将 k 从2 n n 进行压缩到现在还是没有很好的压缩方法，初始化就会很麻烦，时间复杂度会达到 $O(n^4)$
